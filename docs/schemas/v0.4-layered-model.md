# ClawSeat v0.4 — Layered Architecture Schema

> Status: Phase 0 design lock. Binding spec for the TUI engineer and the
> engine refactor (Phase 1+). Any deviation requires review.

## 1. Motivation

Through v0.3 (C8-C16) we built the ledger + event bus assuming `memory` and
`koder` were project-level seats. Operator review clarified they're not:

- **`memory`** is a **machine-wide singleton** (one per computer, shared
  across all projects).
- **`koder`** is an **OpenClaw tenant agent** (one per tenant, bound to at
  most one project at a time). It is never a tmux seat.

This document specifies the three-layer architecture that results, the new
TOML schemas, migration from v1, and the validator contract that the TUI
and engine must both honor.

## 2. The three layers

```
┌─────────────────────────────────────────────────────────┐
│ MACHINE layer (per-computer)                            │
│   Scope: singletons and tenant registry                 │
│   File : ~/.clawseat/machine.toml                       │
│   Runtime: ~/.agents/runtime/memory/ (singleton),       │
│            ~/.openclaw/workspace-<tenant>/              │
│                                                         │
│   Members:                                              │
│     • memory  (exactly 1, claude + api)                 │
│     • openclaw_tenants[N]  (yu, koder, mor, ...)        │
└───────────────────┬─────────────────────────────────────┘
                    │ tenant koder is bound to a project via
                    │ ~/.openclaw/workspace-<tenant>/WORKSPACE_CONTRACT.toml
                    │   .project = "<project-name>"
                    ↓
┌─────────────────────────────────────────────────────────┐
│ PROJECT layer (per-project)                             │
│   Scope: 6 fixed role seats, no memory, no koder        │
│   File : ~/.agents/profiles/<project>-profile-dynamic.toml (v2)  │
│                                                         │
│   Seats (all required, but ancestor+planner minimum):   │
│     • ancestor   — patrol + learning + project monitor  │
│     • planner    — dispatch authority                   │
│     • builder    — implementation (subagent fan-out)    │
│     • reviewer   — code review (subagent fan-out)       │
│     • qa         — verification (subagent fan-out)      │
│     • designer   — visual direction (optional in v2,    │
│                    but declared in seat list)           │
└─────────────────────────────────────────────────────────┘
```

### Seat role enum (exactly these six)

```
ancestor | planner | builder | reviewer | qa | designer
```

Any other string in `seats[]` is a validation error.

## 3. `~/.clawseat/machine.toml` schema (v1)

```toml
# Machine-level ClawSeat configuration.
# Describes singletons (memory) and the tenant registry that projects
# reference via openclaw_frontstage_agent.

version = 1

# ── Machine-singleton services ─────────────────────────────────────────
# Currently only memory. Every project references it via
# project-profile's `machine_services = ["memory"]`.

[services.memory]
role = "memory-oracle"
tool = "claude"                              # ONLY claude supported v1
auth_mode = "api"                            # api | oauth_token (memory never OAuth interactive)
provider = "minimax"                         # anthropic-console | minimax | xcode-best
model = "MiniMax-M2.7-highspeed"             # optional; provider default if omitted
runtime_dir = "~/.agents/runtime/memory"     # singleton runtime home
storage_root = "~/.agents/memory"            # KB + project-scoped entries
launch_args = []
monitor = true

# ── OpenClaw tenant registry ───────────────────────────────────────────
# One entry per OpenClaw tenant that can host a koder agent.
# Each tenant's binding (which project it's currently the frontstage of)
# lives in ~/.openclaw/workspace-<name>/WORKSPACE_CONTRACT.toml.project,
# NOT here. This file only enumerates tenants + their workspace paths.

[openclaw_tenants.yu]
workspace = "~/.openclaw/workspace-yu"
description = "operator ywf's primary install-side tenant"

[openclaw_tenants.koder]
workspace = "~/.openclaw/workspace-<tenant>"
description = "original koder tenant, currently bound to cartooner"

# [openclaw_tenants.mor]
# workspace = "~/.openclaw/workspace-mor"
# description = "third-tenant example, not currently bound"
```

### Validator rules for machine.toml

- `version == 1`
- Exactly one `[services.memory]` table present
- `services.memory.auth_mode` ∈ {`api`, `oauth_token`}
- Every `openclaw_tenants.X.workspace` path exists and contains
  `WORKSPACE_CONTRACT.toml`
- Tenant names match `[a-z][a-z0-9_-]*`

## 4. Project profile schema v2

```toml
# ~/.agents/profiles/install-profile-dynamic.toml (v2)

version = 2

profile_name = "install"
template_name = "gstack-harness"
project_name = "install"
repo_root = "{CLAWSEAT_ROOT}"
tasks_root = "~/.agents/tasks/install"
project_doc = "~/.agents/tasks/install/PROJECT.md"
tasks_doc = "~/.agents/tasks/install/TASKS.md"
status_doc = "~/.agents/tasks/install/STATUS.md"
send_script = "{CLAWSEAT_ROOT}/core/shell-scripts/send-and-verify.sh"
agent_admin = "{CLAWSEAT_ROOT}/core/scripts/agent_admin.py"
workspace_root = "~/.agents/workspaces/install"
handoff_dir = "~/.agents/tasks/install/patrol/handoffs"

# ── NEW in v2: layered dependencies ────────────────────────────────────

# Declare which machine-level services this project consumes.
# Validator asserts each name is defined under [services.X] in machine.toml.
machine_services = ["memory"]

# Declare which OpenClaw tenant agent is this project's frontstage (koder).
# Validator asserts:
#   1. The named tenant exists in machine.toml.
#   2. The tenant's workspace-<name>/WORKSPACE_CONTRACT.toml has
#      project = "<this project_name>".
# If the binding is wrong, validation fails with a clear error and a fix
# command (e.g. `agent-admin project koder-bind --project install --tenant yu`).
openclaw_frontstage_agent = "yu"

# ── Seats: exactly 6 allowed role names, ancestor+planner minimum ─────

seats = ["ancestor", "planner", "builder", "reviewer", "qa", "designer"]

# Roles are fixed 1:1 with seat names in v2. The two are redundant but
# kept for v1 compat; validator asserts seat_roles.X matches the canonical
# expansion.

[seat_roles]
ancestor = "ancestor"
planner = "planner-dispatcher"
builder = "builder"
reviewer = "reviewer"
qa = "qa"
designer = "designer"

# ── Per-seat runtime overrides ────────────────────────────────────────

[seat_overrides.ancestor]
tool = "claude"
auth_mode = "oauth_token"
provider = "anthropic"
# parallel_instances NOT permitted for ancestor (singleton per project).

[seat_overrides.planner]
tool = "claude"
auth_mode = "oauth_token"
provider = "anthropic"
# parallel_instances NOT permitted for planner (singleton per project).

[seat_overrides.builder]
tool = "claude"
auth_mode = "oauth_token"
provider = "anthropic"
parallel_instances = 1                       # default. Max 10. Legal for builder/reviewer/qa only.

[seat_overrides.reviewer]
tool = "codex"
auth_mode = "api"
provider = "xcode-best"
parallel_instances = 1

[seat_overrides.qa]
tool = "claude"
auth_mode = "api"
provider = "minimax"
parallel_instances = 1

[seat_overrides.designer]
tool = "gemini"
auth_mode = "oauth"
provider = "google"
# parallel_instances NOT permitted for designer (singleton per project).

# ── Dynamic roster (carried over from v1) ─────────────────────────────

[dynamic_roster]
enabled = true
session_root = "~/.agents/sessions"
bootstrap_seats = ["ancestor"]                # ancestor boots first per project
default_start_seats = ["ancestor", "planner"] # core loop minimum

# ── Patrol config (ancestor's domain) ─────────────────────────────────

[patrol]
planner_brief_path = "~/.agents/tasks/install/planner/PLANNER_BRIEF.md"
cadence_minutes = 30

# ── Observability ─────────────────────────────────────────────────────

[observability]
announce_planner_events = true
announce_event_types = ["task.completed", "chain.closeout", "seat.blocked_on_modal", "seat.context_near_limit"]

# ── EXPLICITLY REMOVED from v2 ────────────────────────────────────────
# These fields existed in v1 but must NOT appear in v2. Validator rejects.
# - heartbeat_owner         (was "koder"; koder is no longer a project seat)
# - heartbeat_seats         (same reason)
# - heartbeat_transport     (koder uses OpenClaw native transport, not tmux)
# - heartbeat_receipt       (OpenClaw tenant has its own heartbeat)
# - feishu_group_id         (moved to PROJECT_BINDING.toml in C2)
# - any `memory` or `koder` in seats[], seat_roles, materialized_seats,
#   runtime_seats, default_start_seats, bootstrap_seats
```

## 5. PROJECT_BINDING.toml v2 extensions

C2's PROJECT_BINDING.toml stays. Two new fields:

```toml
version = 2                                  # was 1

project = "install"
feishu_group_id = "<FEISHU_GROUP_ID>"
feishu_group_name = "ClawSeat Squad"         # from R6-K2
feishu_external = false                      # from R6-K2
feishu_bot_account = "koder"
require_mention = false
bound_at = "..."

# ── NEW in v2 ────────────────────────────────────────────────────────

openclaw_frontstage_tenant = "yu"            # cross-validated against
                                             # profile.openclaw_frontstage_agent
bound_via = "agent-admin project koder-bind" # audit trail
last_heartbeat_check = "..."                 # optional; set by patrol
```

### Cross-validation

When a project is loaded:
- profile.openclaw_frontstage_agent **must equal** binding.openclaw_frontstage_tenant
- tenant's workspace-<name>/WORKSPACE_CONTRACT.toml.project **must equal** profile.project_name
- Any mismatch → hard validation error with the exact three commands to fix

## 6. Migration v1 → v2

The historical automated migration has completed and the standalone migration
script has been removed. Operators should use `scripts/install.sh --reinstall
<project>` to regenerate a canonical v2 profile.

| v1 state | v2 action |
|---|---|
| `seats` contains `memory` | Remove from `seats`, add `machine_services = ["memory"]`, ensure machine.toml has `[services.memory]` |
| `seats` contains `koder` | Remove from `seats`, set `openclaw_frontstage_agent = <detected tenant>`, validate binding |
| `heartbeat_*` fields | Remove all |
| Missing primary in seats | Regenerate profile with `memory` v2 primary |
| Missing `designer` in seats | Insert `designer` + ask operator (via TUI) tool/auth defaults |
| `builder-1`, `builder-2`, ..., `builder-N` present | Collapse to single `builder`, set `seat_overrides.builder.parallel_instances = N` |
| `reviewer-1`, `reviewer-2`, ... | Same collapse |
| `qa-1`, `qa-2`, ... | Same collapse |
| No ancestor-role skill declared | Inject `ancestor` role; ancestor-cc tmux session becomes a real tracked seat |

Migration outputs a diff report and backs up `<profile>.bak.v1.<ts>` before
writing.

## 7. Validator contract

Published API at `core/lib/profile_validator.py` (Phase 1):

```python
@dataclass
class ValidationResult:
    ok: bool
    errors: list[str]        # blocking
    warnings: list[str]      # informational
    normalized: dict | None  # the validated, canonical form

def validate_profile_v2(path: Path) -> ValidationResult: ...
def validate_machine_config(path: Path) -> ValidationResult: ...
def write_validated(payload: dict, path: Path) -> Path:
    """Validate then atomically write. Raises on failure."""
```

### Rules enforced (rejects on violation)

**Profile**:
1. `version == 2`
2. `seats` ⊆ the template-declared primary seat plus worker roles
3. `seats` ⊇ `{memory, planner}` for v2 minimal (v1 migration may still alias ancestor)
4. `seats` no duplicates
5. `openclaw_frontstage_agent` is a valid key in machine.toml.openclaw_tenants
6. `machine_services` every entry is a valid `services.X` in machine.toml
7. `seat_roles.X` for every X in `seats` maps to the canonical role name
8. No deprecated fields: `heartbeat_*`, `feishu_group_id`, any seat named
   `memory` / `koder` / `builder-N` / `reviewer-N` / `qa-N`
9. `seat_overrides.X.parallel_instances` in `[1, 10]`
10. `parallel_instances > 1` only permitted for X ∈ `{builder, reviewer, qa}`
11. `seat_overrides.X.tool/auth_mode/provider` passes
    `agent_admin_config.validate_runtime_combo`
12. PROJECT_BINDING.toml if exists must cross-validate (section 5)

**machine.toml**:
1. `version == 1`
2. Exactly one `[services.memory]` table
3. Every tenant workspace path exists
4. Tenant names match regex `[a-z][a-z0-9_-]*`
5. `services.memory` has required fields

## 8. Runtime expansion of `parallel_instances`

When `seat_overrides.builder.parallel_instances = 3`:

- ClawSeat instantiates 3 tmux seats: `builder_1`, `builder_2`, `builder_3`
- Each has its own session.toml, workspace, runtime HOME
- state.db rows: 3 entries with `role = "builder"`, distinct `seat_id`s
- `dispatch_task.py --target-role builder` picks least busy across all 3
  (pick_least_busy_seat handles it transparently)
- When a project only has `parallel_instances = 1`, expansion produces a
  single seat named `builder` (no suffix) — matches today's per-role-single
  topology

Naming convention: `{role}` for N=1, `{role}_{n}` for N>1.

## 9. TUI engineer interface

The TUI engineer consumes this document + owns:

| TUI surface | Consumes | Produces |
|---|---|---|
| Install wizard | machine.toml current state, legal enums | v2 profile TOML (piped through `write_validated`) |
| Machine view | `machine_config.load()` | modifications via `machine_config.update()` |
| Project seat view | `state.list_seats(project, normalize_parallel=True)` | no writes (read-only view) |
| koder-bind wizard | `machine_config.list_openclaw_tenants()` | calls `project_binding.bind_koder(project, tenant)` |
| parallel_instances editor | profile, state.db current count | writes `seat_overrides.X.parallel_instances` |

My engineering provides:
- `profile_validator.write_validated(d, path)`
- `machine_config.load_machine() -> MachineConfig`
- `machine_config.list_openclaw_tenants() -> list[Tenant]`
- `machine_config.validate_tenant(name) -> tuple[bool, str]`
- `project_binding.bind_koder(project, tenant) -> None`
- `state.list_seats(project, normalize_parallel=True)`

### Integration test at the seam

## 10. Open design questions (require operator decision)

1. **Does install project need `designer`?**
   Current install has no designer. Default: YES (6 seats minimum). Operator
   can disable per seat via `dynamic_roster.bootstrap_seats` / `default_start_seats`
   exclusion.

2. **`parallel_instances` default for fan-out worker roles**: 1 or 2?
   Proposal: 1. Operators opt-in to more when rate limits or work volume demand it.

3. **primary tmux session rename**: current legacy name is generic, not
   project-prefixed. Should it be `install-memory` to match the
   `<project>-<seat>` convention? Proposal: YES, now covered by v2 minimal.

4. **Cartooner's designer**: currently at `cartooner-designer-1-gemini`. After
   migration: `cartooner-designer-gemini` (drop the -1 since it's singleton).

## 12. Non-goals for v0.4

- Reworking `openclaw.json` schema (out of ClawSeat scope)
- Replacing tmux as the per-seat runtime (still the simplest isolation layer)
- Automating `claude setup-token` refresh (blocked on upstream #47754)
- Cross-machine seat topology (stays single-machine)
- Replacing the event bus with Kafka/NATS (overkill for the use case)
